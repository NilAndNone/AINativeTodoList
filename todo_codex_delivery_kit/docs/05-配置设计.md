# 05 配置设计

## 1. 本机运行时配置

用途：告诉代码仓“数据仓在哪”。

建议路径：

```text
~/.config/ainative-todo/config.toml
```

建议内容：

```toml
data_repo = "/ABS/PATH/TO/AINativeTodoData"
profile = "default"
```

### 设计原则

- 不进 git
- 只放机器本地信息
- 不放项目映射，不放工作流策略

## 3. Codex 项目配置建议

路径：

```text
<CODE_REPO>/.codex/config.toml
```

注意：这是**示例**。真正使用时要结合你本机路径。

```toml
model = "gpt-5.4"
approval_policy = "on-request"
sandbox_mode = "workspace-write"

[sandbox_workspace_write]
network_access = false
writable_roots = ["/ABS/PATH/TO/AINativeTodoData"]

[mcp_servers.ainative_todo]
command = "python3"
args = ["-m", "ainative_todo_service.mcp_server"]
required = true
enabled = true
```

### 为什么建议这样配

- `approval_policy = "on-request"`：你要看 diff 后确认，不适合无脑 `never`
- `workspace-write`：够用且比 full access 稳
- `writable_roots`：把数据仓显式放进可写目录

## 4. skill 放置位置

Codex repo skill 建议放：

```text
<CODE_REPO>/.agents/skills/todo-local-workflow/
```

内容至少包括：

- `SKILL.md`
- `references/`
- 可选 `agents/openai.yaml`

---

## 5. 为什么配置不用 YAML

不是 YAML 不行，而是：

- 你现有项目是 Python
- Python 标准库有 `tomllib`
- 少一个依赖，就少一层泥潭

在“个人长期维护系统”里，依赖少一点，通常比语法华丽更重要。
