# 06 人类工作流

这是给你自己看的极简版，不是给 agent 看的。

## 开工

你对 Codex 说：

```text
开工，按 todo workflow 处理今天的工作台。
```

预期行为：

1. agent 读取 overview
2. 若 today 未归档，提醒你
3. 生成 `today.md` 的 preview diff
4. 你确认
5. agent apply
6. agent 摘要今日必须项 / blocked / overdue

## 白天更新

### 更新既有任务

你说：

```text
把 UTC 那个性能优化任务改成 blocked，备注改成等依赖方接口。
```

预期行为：

1. agent 找任务
2. 若有歧义，列候选
3. 若缺字段，不追问无关信息
4. 生成 diff
5. 你确认
6. agent apply

### 新增任务

你说：

```text
新增一个 MISC 任务，明天下午前整理周会纪要。
```

预期行为：

1. agent 补齐缺字段
2. 生成 diff
3. 你确认
4. agent apply

## 周期总结

你说：

```text
生成本周周报。
```

预期行为：

1. 生成 report preview
2. 你确认
3. 落盘
4. agent 摘要本周完成 / 风险 / 延期 / 下阶段关注点

## 收工

你说：

```text
收工。
```

预期行为：

1. agent 读取 today 与 pending changes
2. 列出归档摘要 + diff
3. 你确认
4. apply
5. 自动在数据仓 `git commit + push`
6. 告诉你归档路径和提交结果

## 你的最低操作成本目标

理想状态是：

- 你主要发自然语言
- 看 diff
- 点头确认
- 偶尔手改数据仓

系统不应该逼你天天自己去搓 CSV 或手写 Markdown 表格。那样就不是 todo 系统，那是某种办公室 roguelike。
