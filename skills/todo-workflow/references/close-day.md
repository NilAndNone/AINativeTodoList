# 收工归档

## 流程

1. **读取 `today.md`**，检查明显遗漏：
   - 状态为空的任务行
   - `临时新增` 中缺少必要字段（标题、项目、优先级、状态）

2. **`doing` 任务可跨天延续**，不强制要求写入"实际完成"或"未完成 & 原因"。

3. **提醒用户确认**——列出将要同步的变更摘要，等待用户明确确认。

4. **用户确认后调用命令**：
   ```bash
   python3 scripts/todo_workflow.py close-day
   ```

5. **检查命令输出**：
   - 成功：告知用户归档路径（`daily/YYYY-MM/WXX/YYYY-MM-DD.md`）。
   - 校验错误：报告具体错误信息，协助用户修正 `today.md` 后重试。

## 约束

- **不自动执行 close-day**，必须用户确认后才能调用。
- 不修改 `data/tasks.csv` 的不可变字段（标题、项目、优先级、创建日期、DDL、产出）——这由脚本校验保证。
