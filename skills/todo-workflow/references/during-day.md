# 白天执行

## 操作映射

| 用户说 | Agent 动作 |
| --- | --- |
| 口头告知进展 | 编辑 `today.md` 对应行的 `状态` 和 `备注` |
| 要求新增任务 | 填写 `临时新增` 表格 |
| 完成了某任务 | 填写 `实际完成` 表格 |
| 任务没做完 | 填写 `未完成 & 原因` 表格 |

## 不可变字段约束

对来自 `data/tasks.csv` 的已有任务，**只允许更新** `状态` 和 `备注`。

以下字段不可修改（收工同步时会按 `data/tasks.csv` 校验）：

- 标题
- 项目
- 优先级（P）
- 创建日期
- DDL
- 产出

## today.md 固定结构

```markdown
# YYYY-MM-DD 周X

## 今日必须完成
| ID | 标题 | 项目 | P | 创建 | DDL | 产出 | 状态 | 备注 |

## 今日计划（非必须）
| ID | 标题 | 项目 | P | 创建 | DDL | 产出 | 状态 | 备注 |

## 临时新增
| 标题 | 项目 | P | DDL | 产出 | 状态 | 备注 |

## 实际完成
| ID | 标题 | 完成情况 |

## 未完成 & 原因
| ID | 标题 | 原因 | 后续计划 |

## 备注
```

## 表头定义

### 今日必须完成 / 今日计划（非必须）

| 列 | 说明 |
| --- | --- |
| ID | 任务 ID，格式 `项目-YYYYMMDD-NN` |
| 标题 | 任务标题 |
| 项目 | 项目代码（UTC / HYP / TAR / HYE / AGE / MISC） |
| P | 优先级（P0 / P1 / P2 / P3） |
| 创建 | 创建日期 YYYY-MM-DD |
| DDL | 截止日期 YYYY-MM-DD，可为空 |
| 产出 | 预期交付物 |
| 状态 | todo / doing / blocked / done / cancelled |
| 备注 | 自由文本 |

### 临时新增

无 ID 列——收工时由脚本自动分配 ID 并写入 `data/tasks.csv`。

### 实际完成

通过 ID 或标题匹配任务，收工时标记为 `done`。

### 未完成 & 原因

记录未完成的任务及原因，供收工归档和周期总结引用。
